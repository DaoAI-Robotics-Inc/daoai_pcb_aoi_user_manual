模型管理
=================

系统的检测能力由一组深度学习模型驱动。本章介绍如何管理这些模型与其训练数据集：包括查看 / 上传 / 切换模型，以及导入导出数据集、查看训练分布并发起训练。

模型有两个入口：

- **设置 → 管理模型**：模型清单与生命周期操作（设为默认、重命名、导出、删除等）。
- **模型页面（Models）**：以数据集为中心的视图，用于查看训练样本、训练 / 测试分布并发起训练。

模型清单（管理模型）
-----------------------

在 **设置** 左侧点击 **管理模型**，可按 **全部 / 预训练 / 已训练** 筛选系统中的模型。

.. image:: images/manage_models.png
   :scale: 60%
   :alt: 管理模型界面示意

常用操作：

- **上传模型**：导入外部训练好的模型文件。
- **训练模型**：基于当前数据集发起一次训练。
- **设为默认**：将某模型设为对应检测类型的默认推理模型。
- **重命名 / 导出 / 删除**：对模型条目进行管理。
- **导出数据集 / 导入数据集**：在不同环境间迁移训练样本。
- **重置初始图片**：将数据集恢复到初始样本集合。

数据集与训练（模型页面）
---------------------------

模型页面以 **数据集** 为中心，展示当前检测类型已积累的训练样本（来自自动编程、评估与现场反馈）。

.. image:: images/models_page.png
   :scale: 60%
   :alt: 模型数据集与训练界面示意

- 顶部在 **数据集** 与 **训练** 视图间切换。
- 右侧显示 **训练集 / 测试集** 的样本数量与划分比例（训练 / 测试分布）。
- **查看训练 / 测试分布**：检查样本在各类别上的分布是否均衡，避免某一类别样本过少。
- **导入数据集 / 导出数据集**：在环境间迁移或备份样本。

训练流程
-----------------

1. 确认数据集中已积累足够且类别均衡的样本（可先在产品编程页面通过反馈补充样本）。
2. （可选）点击 **查看训练 / 测试分布**，确认各类别样本数量合理。
3. 点击 **训练模型**，系统基于当前数据集发起训练（耗时视数据量与硬件而定）。
4. 训练完成后，可在 **管理模型** 中将新模型 **设为默认**，使其在后续检测中生效。

.. note::
   反馈写入数据集后，建议在稳定的光照与工艺条件下积累一段时间样本再训练，避免短期波动导致模型过度拟合。
